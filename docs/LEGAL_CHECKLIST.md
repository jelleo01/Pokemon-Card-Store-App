# 이용약관 / 개인정보 체크리스트

> 핸드폰 번호를 받기 때문에 **개인정보처리방침은 필수**. 약관/방침 게시 없이 가입받으면 개인정보보호법 위반.

---

## 1. 이용약관 (Terms of Service)

### 필수 포함 사항
- [ ] 서비스 명칭, 운영자 정보
- [ ] 회원가입/탈퇴 절차
- [ ] 회원의 의무 (허위 정보 금지, 타인 명의 도용 금지)
- [ ] 운영자의 권리/의무
- [ ] 게시물 삭제 기준 (욕설, 광고성, 매장 정보 위변조)
- [ ] 서비스 중단/변경 가능성
- [ ] 분쟁 해결, 관할 법원 (보통 서울중앙지방법원)
- [ ] 면책 조항
- [ ] 약관 변경 시 공지 방법 (앱 내 공지 + 30일 전 고지)

### 추천 템플릿
- 표준약관: 공정거래위원회 [표준약관 검색](https://www.ftc.go.kr) → "전자상거래 표준약관"
- 무료 생성: [이지로 (eajiro)](https://easyrro.com), [터미널리 (Termly)](https://termly.io)
- 비슷한 서비스 약관 참고 (당근마켓, 번개장터 등)

## 2. 개인정보처리방침 (Privacy Policy) — **필수**

### 수집 항목
- [ ] **필수**: 핸드폰 번호 (인증/로그인)
- [ ] **필수**: 트레이너 ID (식별)
- [ ] **선택**: 위치 정보 (현재 위치 사용 시) — Geolocation
- [ ] **선택**: 활동 지역 (시/구) — 사용자가 직접 입력
- [ ] **자동**: IP, User-Agent, 쿠키, 가입/로그인 일시

### 수집 목적
- 본인확인 / 부정가입 방지
- 서비스 제공 (커뮤니티 글 작성/조회)
- 위치 기반 매장 추천
- 부정행위 차단

### 보유 기간
- 회원 정보: **회원 탈퇴 시까지**
- 핸드폰 번호: 탈퇴 후 30일까지 (재가입 어뷰징 방지) → 그 후 영구 삭제
- 위치 정보: 사용 시점에만 메모리에 보유, DB 저장 X (또는 사용자가 명시 저장 시만)
- 게시글/댓글: 삭제 시까지 (탈퇴해도 글은 익명 처리하고 유지 가능 — 약관에 명시)

### 제3자 제공
- 없음 (또는 SMS 인증 위해 Twilio에 핸드폰 번호 전달)

### 처리 위탁
- [ ] **Supabase** (DB, 인증) — 위치: 사용자가 선택한 region (예: AWS Asia Northeast Seoul)
- [ ] **Twilio** (SMS 인증) — 위치: 미국 (해외 이전 동의 필요)
- [ ] **카카오** (지도 API) — 좌표 정보 전달
- [ ] **Vercel/Netlify/Cloudflare** (호스팅) — 위치, IP 로그

### 사용자 권리
- 열람 / 정정 / 삭제 / 처리정지 요청
- 동의 철회
- 개인정보 이동권 (GDPR 영향)

### 개인정보 보호 책임자
- 이름 / 이메일 (필수 명시)

### 추천 템플릿
- 한국인터넷진흥원 [개인정보보호 종합포털](https://www.privacy.go.kr) → 자가진단 → 개인정보처리방침 작성 가이드

## 3. 가입 시 동의 UI

### 필수 동의 (체크박스 따로따로)
- [ ] 이용약관 동의 (필수)
- [ ] 개인정보처리방침 동의 (필수)
- [ ] 만 14세 이상 확인 (필수) — 14세 미만은 법정대리인 동의 필요

### 선택 동의
- [ ] 마케팅 정보 수신 동의 (이메일/SMS) — 선택
- [ ] 위치 기반 서비스 동의 — 선택 (카카오 Geolocation 사용 시)
- [ ] 푸시 알림 동의 (POST-MVP)

### 구현 패턴 (Login STEP 1)
```tsx
<Checkbox required>
  <a onClick={() => setShowTerms(true)}>이용약관</a>에 동의합니다 (필수)
</Checkbox>
<Checkbox required>
  <a onClick={() => setShowPrivacy(true)}>개인정보처리방침</a>에 동의합니다 (필수)
</Checkbox>
<Checkbox required>
  만 14세 이상입니다 (필수)
</Checkbox>
<Checkbox>
  마케팅 정보 수신에 동의합니다 (선택)
</Checkbox>
```

DB 저장:
```sql
-- profiles 테이블
terms_agreed_at        timestamptz NOT NULL,
privacy_agreed_at      timestamptz NOT NULL,
age_confirmed_at       timestamptz NOT NULL,
marketing_opt_in       boolean DEFAULT false,
marketing_agreed_at    timestamptz,
```

## 5. 포켓몬 IP 관련

> **주의**: 포켓몬은 The Pokémon Company의 등록상표.

### 안전한 사용
- ✅ "포켓몬 카드 트레이딩 정보를 공유하는 커뮤니티" 같은 일반적 명시
- ✅ 사용자가 작성한 글 안에 자유롭게 등장
- ✅ 픽셀 스타일의 일반 아이콘 (몬볼 모양, 카드 모양 등)

### 위험한 사용
- ❌ 공식 포켓몬 캐릭터 이미지/스프라이트 사용
- ❌ "포켓몬" 로고 그대로 사용
- ❌ "포켓몬"을 앱 이름에 직접 사용 ("PokemonCardKorea" 같은 거)
- ❌ 공식 포켓몬 카드 이미지 호스팅

### 권장
- 앱 이름은 일반화 (예: "트레이너스 / Trainers", "카드덱 / CardDek", "포켓 / Pocket")
- 로고는 자체 디자인 (몬볼 비슷하지만 자체 일러스트)
- 약관에 "당사는 The Pokémon Company와 무관합니다" 명시

## 6. 신고/차단 정책

### 신고 대상
- 욕설, 비방
- 광고성 글 (외부 판매 링크 등)
- 매장 허위 정보
- 도배

### 처리 흐름
1. 사용자가 신고 → DB `reports` 테이블에 INSERT
2. 관리자 페이지에서 검토
3. 처리 결과:
   - 게시글 숨김 / 삭제
   - 작성자 경고 / 일시 정지 / 영구 차단
   - 기각

### MVP에서는
- 신고 버튼만 제공 (처리는 관리자 수동)
- 관리자 페이지: Supabase Dashboard에서 직접 SQL로 처리하거나, 간단한 admin 페이지 제작

## 7. 데이터 삭제 / 회원 탈퇴

### 회원 탈퇴 시
- `auth.users` 행 삭제 → CASCADE로 `profiles`, `posts`, `comments`, `hearts` 모두 삭제
- 또는 **soft delete**: `profiles.deleted_at` 만 채우고, 글은 익명 처리 ("탈퇴한 트레이너")
  - 추천: 글이 커뮤니티 자산이므로 삭제보다 익명화가 나음 → 약관에 명시 필요

### 데이터 삭제 요청 (사용자가 직접)
- 30일 내 처리 (개인정보보호법)
- 처리 결과 통지
